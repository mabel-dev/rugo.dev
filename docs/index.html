Why Rugo

Thin by design. Opinionated about what not to load.

Rugo was built as the file layer for the Opteryx SQL engine, where the constraint was simple: read as little data as possible, hold as little in memory as possible. Column projection and row-group pruning happen before any decoding. The result is a library that is fast on selective queries, tiny on disk, and carries no PyArrow, Pandas, or NumPy into your environment.

⚡

Serverless-first

AWS Lambda and GCP Cloud Functions bill by the millisecond. A 137 ms PyArrow import adds real cost to every cold start. Rugo imports in 2.6 ms — over 50× faster.

📦

Tiny container images

At 16× smaller than PyArrow, rugo meaningfully shrinks image size, speeds scale-out, and keeps you well clear of AWS Lambda's 250 MB unzipped layer limit.

🔍

Read less, go faster

Column projection and row-group pruning are first-class citizens. Skip the columns you don't need. Skip the row groups that can't match. Rugo's advantage grows with selectivity.

🔒

No surprise dependencies

The wheel bundles everything it needs — Draken, the columnar substrate, ships inside. pip install rugo is the entire dependency story.

Where PyArrow is faster: bulk full-table decoding. PyArrow's Arrow C++ engine is highly optimised for raw throughput, and that is a decade of effort rugo does not try to replicate. If you are streaming entire large tables into memory as fast as possible, PyArrow is the right tool. Rugo is opinionated in the other direction: it is built to read less — push filters early, skip columns, prune row groups — so that less data ever enters memory in the first place. If that trade-off fits your workload, or you simply cannot afford 124 MB of Arrow C++ in your dependency tree, rugo is for you.

Quickstart

One install, three formats.

Rugo reads and writes Parquet, CSV, and JSONL. The API is the same shape across all three: pass a path or bytes, get columnar data back.

pip install rugo 
from rugo import parquet

# Stream a projected, row-group-pruned read — no PyArrow, no NumPy
with parquet.read_parquet(
    "planets.parquet",
    columns=["id", "name"],
    filters=[("id", ">", 4)],
) as reader:
    for morsel in reader:
        print(morsel.column(b"name").to_pylist())

# Write a Morsel to Parquet bytes
data = parquet.write_parquet(morsel, compression="zstd")
from rugo.csv import read_csv, write_csv

result = read_csv("data.csv", columns=["name"], predicates=[("age", ">", 30)])
if result["success"]:
    for vec in result["columns"]:
        print(vec.to_pylist())
from rugo.jsonl import read_jsonl, write_jsonl

result = read_jsonl("events.jsonl", columns=["id", "status"],
                   predicates=[("status", "==", "active")])
print(result["num_rows"], "rows matched")

A complete worked example on a real dataset is available in Google Colab or as a notebook on GitHub.

Formats

Parquet, CSV, JSONL — read and write.

All three formats support column projection and predicate filtering. Writers produce standard output readable by any compliant tool.

FunctionReturns
read_parquet(source, columns, filters)context manager → one Morsel per row group
read_metadata(source)ParquetMetadata (num_rows, schema_columns)
write_parquet(morsel, compression)bytes
write_parquet_with_bounds(morsel)(bytes, {col: (min, max)})

source is a filename or bytes/bytearray/memoryview. filters is a list of (column, op, value) tuples; pruning is at row-group granularity. Ops: = == != < <= > >= in "not in".

Decode: int32/64, float32/64, boolean, byte_array · UNCOMPRESSED, SNAPPY, ZSTD · PLAIN, RLE_DICTIONARY, DELTA_BINARY_PACKED, DELTA_BYTE_ARRAY

Write: INT8–64, FLOAT32/64, BOOL, VARCHAR, DATE32, TIME, TIMESTAMP64, INTERVAL, DECIMAL, ARRAY — ZSTD or uncompressed, per-column min/max stats, optional bloom filters.

Rugo's Parquet advantage is reading less, not raw scan throughput. Row-group pruning and column projection reduce what is decoded; PyArrow's bulk scan is faster when all columns and rows are needed. 

from rugo import parquet

# Metadata only — no column decode
meta = parquet.read_metadata("planets.parquet")
print(meta.num_rows)
print([c.name for c in meta.schema_columns])

# Selective streaming read
with parquet.read_parquet(
    "planets.parquet",
    columns=["id", "name"],
    filters=[("id", ">", 4)],
) as reader:
    for morsel in reader:
        names = morsel.column(b"name").to_pylist()

# Write (ZSTD default)
data = parquet.write_parquet(morsel)
with open("out.parquet", "wb") as f:
    f.write(data)

On narrow files PyArrow is faster across the board — the crossover is driven by how many columns can be skipped and how many rows are eliminated before the typed column build.

# Projected + filtered read
result = read_csv(

if result["success"]:

# TSV variant

Advantage grows with selectivity — PyArrow reads all columns regardless of projection.

from rugo.jsonl import get_jsonl_schema, read_jsonl, write_jsonl

# Infer schema from sample rows
schema = get_jsonl_schema("events.jsonl", sample_size=5)

# Projected + filtered read
result = read_jsonl(
    "events.jsonl",
    columns=["id", "status"],
    predicates=[("status", "==", "active")],
)
if result["success"]:
    print(result["num_rows"], "rows matched")
    for vec in result["columns"]:
        print(vec.to_pylist())

# Read from bytes
with open("events.jsonl", "rb") as f:
    result = read_jsonl(f.read())

# Write a Morsel to JSONL bytes
jsonl_bytes = write_jsonl(morsel)
Data Model

Vectors and Morsels.

Rugo speaks Draken — a bundled columnar substrate that ships inside the wheel. There are two objects you will encounter.

Vector

A single typed column. Call vector.to_pylist() to get a Python list of values. 

vec = morsel.column(b"name")
values = vec.to_pylist()
# ["Mercury", "Venus", ...]

Note: column keys are bytes, not strings.

Morsel

A batch of rows across several columns — a chunk of a table. One Parquet row group produces one Morsel. 

for morsel in reader:
    print(len(morsel), "rows")
    vec = morsel.column(b"id")

A read → write round-trip works naturally: read_parquet yields Morsels, write_parquet / write_csv / write_jsonl all consume a Morsel. 

from rugo import parquet
from rugo.csv import write_csv
from rugo.jsonl import write_jsonl

with parquet.read_parquet("planets.parquet") as reader:
    for morsel in reader:
        csv_bytes   = write_csv(morsel)
        jsonl_bytes = write_jsonl(morsel)
        pq_bytes    = parquet.write_parquet(morsel)
